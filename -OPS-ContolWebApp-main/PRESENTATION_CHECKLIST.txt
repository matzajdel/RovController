🎯 Q&A PREPARATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Q: "Czy to działa z prawdziwym robotem?"
A: Tak! Backend subskrybuje /joint_states z ROS2. 
   Pokazane demo działa standalone, ale na robocie wystarczy 
   uruchomić i automatycznie się połączy.

Q: "Co to jest IK solver?"
A: Inverse Kinematics - oblicza kąty jointów dla zadanej 
   pozycji end-effectora. Backend endpoint gotowy (/robot/ik_solve),
   frontend integration TODO.

Q: "Jakie inne funkcje można dodać?"
A: - Collision detection
   - Path planning visualization
   - Recording/playback
   - Multi-robot support
   - Force feedback display

Q: "Czemu React Three Fiber zamiast czystego Three.js?"
A: R3F daje deklaratywny API w React, łatwiejszy state management,
   i lepszą integrację z resztą aplikacji (routing, hooks).

Q: "Jak testowaliście bez robota?"
A: Mock joint publisher (mock_joint_publisher.py) publikuje 
   sinusoidalne ruchy na /joint_states. Test API script weryfikuje
   wszystkie endpointy.
